Architecture Comparison / ArcheAge

ArcheAge

Modified CryEngine Sandbox MMO

A heavily modified CryEngine 3 MMO with non-instanced open-world housing, multi-continental seamless zones, a player-driven trade economy with physical cargo transport, and a zone-based PvP cycle that transitions between peace, conflict, and war states server-side.

Overview

ArcheAge is a sandbox MMORPG designed by Jake Song, the creator of Lineage, and built by XL Games in South Korea. Development started in 2006, and the game launched in Korea in January 2013 before reaching Western markets through Trion Worlds in September 2014. What makes ArcheAge architecturally relevant is how it pushed CryEngine 3 far beyond its intended purpose.

CryEngine was built for first-person shooters with tight player counts and relatively small maps. XL Games adapted it into an engine capable of running a persistent MMO with two full continents, a shared ocean, non-instanced player housing, physics-based naval combat, a real-time trade economy with physical cargo hauling, and zone-based PvP state cycling. The modifications were extensive enough that the engine's original optimization characteristics broke down in several areas, particularly around large-scale PvP and client rendering of player-dense areas.

The game's server architecture follows a traditional monolithic MMO design where a single logical server manages all zones across both continents and the ocean. Zones cycle independently between peace, conflict, and war states based on PvP activity. The housing system persists player-placed structures in the open world without instancing, creating a finite land economy where ownership is contested, taxed, and sometimes demolished by enemy factions.

ArcheAge went through three Western publishers (Trion Worlds, Gamigo, Kakao Games), launched the buy-to-play "Unchained" variant, ran multiple fresh-start servers, and ultimately shut down Western service in June 2024. Through all of these transitions, the core server architecture remained largely the same modified CryEngine 3 stack that XL Games built in the late 2000s.

Server Infrastructure

ArcheAge follows the traditional Korean MMO server model: a single monolithic server process manages the entire game world for each logical server (called a "world" or "server" by players). This process handles entity simulation, physics, AI, zone state transitions, and player networking. Unlike New World's distributed hub system, everything runs within one process boundary.

CryEngine 3 Modifications

XL Games licensed CryEngine in the CryEngine 2 era and began adapting it for MMO use. During development they incorporated elements from CryEngine 3, particularly the physics engine and animation system, while keeping much of the rendering pipeline on the CryEngine 2 / DX9 path. The result is a hybrid engine that offers both DirectX 9 and DirectX 11 rendering, though the DX9 path was generally considered more stable and performant because the engine was originally coded against it.

The modifications for MMO use were substantial. CryEngine's networking layer (CryNetwork) was designed for small-player-count FPS games, not thousands of simultaneous connections. XL Games had to rework the networking stack, add persistent world state management, build a housing system with physical placement and collision, implement a multi-zone world with seamless transitions, and add the ocean and naval systems. These modifications came at a cost: "the meaning of optimization faded a lot," as community analyses of the engine have noted.

Publisher-Side Infrastructure

A distinctive feature of ArcheAge's architecture is the split between the developer (XL Games) and the publisher. XL Games provides the game server binaries and patch content. The publisher operates the actual server hardware, login servers, payment systems, and customer support infrastructure. This meant the game ran on different hosting setups depending on region:

  • Korea: XL Games operated directly, using Korean data centers
  • Russia: Mail.ru Games hosted with a conservative server count that kept populations dense
  • NA/EU (2014-2019): Trion Worlds operated on their own data center infrastructure
  • NA/EU (2019-2021): Gamigo (who acquired Trion) continued on the same infrastructure
  • NA/EU (2021-2024): Kakao Games took over publishing, migrating to their own hosting

Each publisher transition involved migrating player databases and account systems while keeping the XL Games server binaries essentially the same. The game logic did not change between publishers, but server performance, anti-cheat enforcement, and update cadence varied significantly.

Network & Protocols

ArcheAge's networking is built on CryNetwork, the networking module that ships with CryEngine. CryNetwork uses UDP as its primary transport protocol for game data, with a custom reliability layer built on top for ordered and reliable delivery of critical events.

Aspect-Based Replication

CryNetwork uses an "aspect" system for entity replication. Each networked entity (player characters, NPCs, ships, trade packs, housing objects) can have up to 32 independent aspects. Each aspect represents a group of related data fields, like position/rotation, health/mana, equipment, or buff states. When a particular aspect is marked "dirty" by game code, only that aspect gets serialized and sent to relevant clients. This avoids retransmitting the entire entity state on every change.

The recommendation from Crytek's documentation is to group variables by rate of change. Position and rotation data, which change every frame for moving entities, go in one aspect. Equipment data, which changes rarely, goes in another. This ensures that frequent updates (like movement) do not carry the overhead of infrequent data (like what hat you are wearing).

Channel Architecture

Each connected player has a "channel" to the server. The server maintains one channel per client. All communication flows through these channels. CryNetwork runs its network I/O on a separate thread from the game simulation, using a global lock to synchronize access. To minimize lock contention, most cross-thread calls are queued: game-to-network calls go into a FROM_GAME queue, and network-to-game calls go into a TO_GAME queue.

Player Collision

Unlike most MMOs, ArcheAge uses player collision. Your character can be physically blocked by other players and NPCs. This has significant networking implications because collision detection must be server-authoritative to prevent desync, and it creates unique gameplay situations like physically blocking trade routes or doorways. The physics overhead of simulating thousands of player-to-player collision checks was one factor limiting server capacity.

Physics as a Feature and a Bottleneck

ArcheAge's physics engine was inherited from CryEngine and became both a signature feature and a performance constraint. Ships rock on ocean waves. Trade packs have physical presence. Player collision blocks movement. Siege weapons follow ballistic trajectories. All of this requires continuous physics simulation on the server, which consumes CPU cycles that other MMOs do not spend. When hundreds of players gathered for castle sieges or trade route ambushes, the physics overhead combined with entity replication created severe server-side lag.

Data & Persistence

Housing Persistence

ArcheAge's housing system is one of the most complex persistence challenges in any MMO. Houses and farms are not instanced. They exist in the open world, visible to every player, occupying real space on the map. The server must track the position, rotation, type, ownership, tax status, furniture placement, storage contents, and permission settings of every single player structure.

Housing zones are designated areas throughout both continents with restrictions on structure size and type. Some zones allow only small farms. Others are designated "luxury housing" areas for larger buildings. The finite nature of these zones creates genuine scarcity: when all plots are claimed, new players simply cannot place a house until someone else loses theirs.

Tax and Demolition System

Housing persistence is tied to a weekly tax system. Players must deposit Tax Certificates (crafted or purchased) to maintain their property. Missing one week incurs a penalty but the house stays. Missing two consecutive weeks triggers demolition: the house disappears and the plot opens for anyone to claim. This system acts as a garbage collector for abandoned properties, preventing permanently occupied land from dead accounts.

When a plot opens, the server runs a claim lottery. A 20-second timer starts, during which any player who attempts to claim the land is added to a participant list. After 20 seconds, a random roll selects a winner, who then has 60 seconds of exclusive placement rights. Players can only participate in one land claim every 10 minutes. All of these restrictions are tracked per account, not per character.

On Auroria, the endgame PvP continent, castle owners can demolish all player houses within a 100-meter radius of their castle center. Demolished houses are returned to the owner via in-game mail as a packaged item that can be replanted elsewhere, so the structure data persists even when the physical placement is destroyed.

Trade Economy Data

The trade system tracks supply and demand per route in real time. Each trade pack has a base value, and the percentage of that value you receive at the destination varies from 130% down to 70% based on recent delivery volume. This price recharges at 5% per hour when no deliveries are made. The server must track delivery counts per route, per time window, and calculate the current percentage for every trade NPC in the world.

Trade packs are physical objects. When a player crafts one, it appears on their back, reduces their movement speed, and can be stolen by other players who kill them. If a pack is dropped (by death or by being placed on a ship that sinks), the pack persists in the world as a lootable object with a decay timer. The server must track the position and ownership state of every trade pack in existence.

Labor Points

Labor Points are an account-wide resource that regenerates over time. Patrons regenerate 10 LP every 5 minutes (both online and offline), up to a cap of 5,000. Free players regenerate 5 LP every 5 minutes, only while online, up to 2,000. Nearly every economic action in the game consumes LP: crafting, gathering, opening loot, turning in trade packs, filing crime reports. The server tracks LP balances and regeneration timers per account, across all characters.

Server Merges and the Land Problem

Server merges in ArcheAge were uniquely painful because of the housing system. Even on low-population servers, most land was still occupied because existing players owned multiple plots. Merging two servers means merging two full sets of housing data into one world with the same finite land supply. XL Games and the publishers never found a clean solution to this. Players on merged servers often lost their homes, and the resulting land rush on the merged server was chaotic and controversial every time it happened.

Scaling & Performance

Zone State Cycling

Each PvP zone on the two main continents cycles independently through states: Peace, then Conflict (levels 1 through 5), then War (level 6), then back to Peace. The cycle is driven by PvP activity. When players kill each other in a zone that is in Conflict state, the zone accumulates points toward the next conflict level. At level 6, the zone enters War state for a fixed duration, during which honor (PvP currency) is awarded for kills and special events trigger in certain zones.

The ocean and Auroria (the northern PvP continent) never enter Peace state. They are permanently in War, meaning open PvP is always active. This design means the server must track per-zone PvP counters, state transition timers, and special event triggers independently for every zone in the game.

Castle Sieges (100v100)

Castle sieges were restructured in the 2.5 update to a 100v100 format. The defending guild fields 60 players initially, with 40 reinforcements arriving after 40 minutes. The attacking guild brings 100 players from the start. Sieges happen weekly on Saturdays for each castle, with guilds bidding gold for the right to attack.

Castles exist only on Auroria and are modular structures that guilds build piece by piece. The siege system tracks the health of every castle structure, the position of siege weapons, the guild membership of every participant, and the bid amounts from all guilds that wanted to attack. This is the highest-stress event in the game from a server perspective: 200 players in a single zone, all using abilities, deploying siege weapons, and destroying structures simultaneously.

Naval Combat and Ocean Systems

The ocean is a shared zone connecting both continents. Ships are player-crafted vehicles with physics simulation: they rock on waves, take damage from cannons, and can be boarded by enemy players. Naval combat involves tracking ship hull health, cannon trajectories (ballistic physics), crew positions on the ship, and cargo (trade packs) stored in the hold.

When a ship sinks, its cargo drops into the water as salvageable objects. Players can dive to recover sunken trade packs. The server must maintain underwater entity positions and physics for these dropped objects alongside the surface-level ship simulation.

Fresh Start Server Architecture

Fresh start servers were a recurring strategy to address population decline and economy inflation. These servers used the same server binary and architecture as existing servers, but started with a clean database: no existing characters, no claimed land, no accumulated wealth. Players could not transfer characters from existing servers to fresh start servers.

The appeal was the "land rush" experience of a new server where everyone competes for housing plots from scratch. Kakao Games launched multiple rounds of fresh start servers (September 2022, February 2023) with temporary XP and drop rate boosts. After a fixed period (typically 3 months), fresh start servers would open character transfers to existing live servers, at which point they typically died as players migrated to the larger population.

Technical Deep Dives

Justice System Infrastructure

ArcheAge has a player-run court system. When a player accumulates 50 or more Crime Points (earned by killing same-faction players or stealing crops), they are sent to trial upon their next PvP death. The server must track Crime Points per character, manage a jury queue of eligible players (level 30+, no infamy points), and orchestrate the trial process.

Jury selection pulls 5 players from the queue, ordered by login time. The defendant can plead guilty for an automatic (reduced) sentence or stand trial. During a trial (up to 10 minutes), the defendant presents their case, the jury deliberates, and each juror votes on a sentence. The sentence range (from 2 minutes to several days of in-game prison time) is calculated from current and lifetime criminal points, weighted by the jury's votes.

Prison is a separate instanced zone with a debuff that disables all combat and mobility skills. Prisoners can work off their sentence through labor activities (which consume Labor Points). The server tracks sentence timers per imprisoned character and reduces the timer based on completed prison activities.

Trade Route Server-Side Tracking

The trade system is one of ArcheAge's most server-intensive features. Each trade pack is a physical entity with position, ownership, and cargo type. The server tracks:

  • Pack crafting: 60 LP cost to create, materials consumed from inventory
  • Pack transport: Entity position updates as the carrier moves, speed reduction applied
  • Pack theft: Ownership transfer on carrier death, physical drop to world
  • Ship cargo: Packs stored in ship hold, linked to ship entity
  • Route pricing: Delivery count per route per time window, 130%-70% price scaling, 5%/hr recharge
  • Turn-in: 60 LP cost, gold/gilda stars awarded based on current route percentage

A single player running trade packs generates a continuous stream of position updates (the pack entity moves with them), plus LP consumption events, plus economic state queries (checking route percentages), plus potential PvP interactions. Multiply this by hundreds of players running packs simultaneously, and the trade system becomes one of the most demanding subsystems on the server.

Unchained vs. Legacy Architecture

ArcheAge: Unchained launched in October 2019 as a buy-to-play version without the free-to-play game's subscription model or cash shop power items. Despite the different business model, Unchained ran on identical server architecture. The difference was entirely in the game database configuration: which items were available in the cash shop, how Labor Points regenerated (all players treated as Patrons), and what restrictions applied to free accounts (there were none, since everyone bought the game).

Kakao Games eventually merged Unchained servers into the Legacy ArcheAge servers in August 2023, citing population decline. The merge involved migrating Unchained player databases into the Legacy server databases and reconciling differences in item availability and progression between the two versions.

Architecture Evolution

2006 Development begins at XL Games. Jake Song (Lineage creator) starts building ArcheAge on CryEngine 2, planning a sandbox MMO with naval combat and open-world housing.
2010 Engine partially migrated to CryEngine 3. Physics engine and animation system updated to CryEngine 3, while rendering remains on the CryEngine 2 DX9 pipeline.
2013 Korean launch (January 15). XL Games opens servers in Korea. Initial server count is conservative. Most servers remain in a crowded state months after launch.
2014 Western launch via Trion Worlds (September 16). Massive demand, severe queue times. Founder packs available since April. The land rush creates immediate housing scarcity on every server.
2015 XL Games announces XLE (XL Engine). An open-source (MIT license) game engine under development, suggesting XL Games was already looking beyond CryEngine for future projects.
2016 Auroria expansion and castle siege rework. 100v100 castle siege format introduced in version 2.5. Endgame PvP continent fully opened with guild castle construction.
2018 Gamigo acquires Trion Worlds (October). Trion Worlds is disbanded. Publishing of ArcheAge NA/EU transitions to Gamigo.
2019 ArcheAge: Unchained launches (October). Buy-to-play version on same server architecture, published by Gamigo.
2021 Kakao Games takes over NA/EU publishing (December 2). Four fresh start servers launched simultaneously. Tens of thousands of bot accounts banned. Over 200 billion in-game currency removed.
2023 Unchained merged into Legacy servers (November/December). Announced in August, the actual merge completed November 30. Population too low to sustain separate server versions. One NA server and one EU server remain.
2024 Western servers shut down (June 27). Kakao Games announces closure of all NA/EU ArcheAge servers. ArcheAge Chronicles (sequel on UE5) announced at PlayStation State of Play.
