Overview

Call of Duty: Warzone is a free-to-play battle royale that originally launched in March 2020 as a companion to Modern Warfare (2019). It dropped 150 players onto Verdansk, a massive open-world map, in squads of up to four. Players loot weapons, buy equipment at Buy Stations using in-match cash, fight through the Gulag for a second chance, and battle to be the last squad standing as a gas circle closes in.

The game runs on a heavily modified version of the IW engine, descended from id Tech 3 through years of Infinity Ward iteration. The networking backbone is provided by Demonware, an Activision subsidiary based in Dublin that has powered Call of Duty's online infrastructure since 2005. Demonware handles matchmaking, player profiles, statistics, leaderboards, inventory, and state synchronization using their proprietary "State Engine" (C++) and "Matchmaking+" platforms, with core services written in Erlang and Python.

Warzone's architecture is shaped by several contentious design decisions: a 20 Hz server tick rate for battle royale (vs. 60 Hz for regular multiplayer), SBMM (Skill-Based Matchmaking) that has driven community controversy for years, and the RICOCHET anti-cheat system that operates at the kernel level to combat an arms race with cheat developers. The game integrates tightly with each annual Call of Duty release, sharing progression systems, weapon pools, and backend infrastructure.

IW Engine

A proprietary engine evolved from id Tech 3 (Quake III Arena). Modern versions support photogrammetry-based rendering, real-time ray tracing, and massive open-world maps. The engine handles both the 6v6 multiplayer modes and the 150-player battle royale, with the networking layer adapted to handle radically different player densities.

Demonware Backend

Founded in 2002 in Dublin, acquired by Activision in 2007. Demonware provides the State Engine for netcode, Matchmaking+ for lobby creation, and handles stats, leaderboards, storage, profiles, paintjobs, gunsmith data, and inventory. Their main server infrastructure is written in Erlang and Python, with offices in Dublin, Vancouver, Los Angeles, and Shanghai.

26+ Server Locations

Dedicated game servers span major cities worldwide: New York, LA, Chicago, Dallas, Miami, Toronto, Montreal in North America; London, Paris, Frankfurt, Amsterdam in Europe; Tokyo, Seoul, Singapore, Mumbai in Asia; Sydney in Oceania; Sao Paulo in South America; plus Middle East servers added in 2024. Warzone and multiplayer servers are kept on separate infrastructure.

RICOCHET Anti-Cheat

A multi-layer anti-cheat system combining a kernel-level PC driver, server-side machine learning detection, in-game mitigations (Damage Shield, Cloaking, Disarm, Hallucinations), phone verification, hardware bans, and TPM 2.0/Secure Boot verification. The kernel driver only runs while the game is active, unlike always-on alternatives.

Server Infrastructure

Warzone runs entirely on dedicated servers for battle royale modes. Regular 6v6 multiplayer uses a hybrid approach where public matches typically use dedicated servers but private/custom matches may fall back to player-hosted (P2P) sessions. The dedicated server fleet is managed by Demonware, operating from data centers across 26+ locations worldwide.

Data Center Distribution

Server locations are concentrated in high-population areas. North America has the densest coverage with servers in New York, Los Angeles, Chicago, Dallas, Miami, Toronto, and Montreal. Europe is served from London, Paris, Frankfurt, and Amsterdam. Asia-Pacific has servers in Tokyo, Seoul, Singapore, Mumbai, and Sydney. South America runs from Sao Paulo, and Activision added dedicated Middle East servers in 2024 to improve service for that region and parts of Africa.

Warzone and regular multiplayer operate on separate server infrastructure. This separation prevents a new multiplayer title launch from degrading Warzone performance and allows independent scaling for each mode's demand patterns.

Demonware's Technology Stack

Demonware's core infrastructure runs on Erlang for its concurrent networking services and Python for tooling and automation. The State Engine provides high-performance state synchronization in C++, handling the netcode that game servers use to keep clients in sync. Matchmaking+ operates as a service layer above the game servers, handling lobby creation, skill evaluation, region routing, and queue management.

Demonware also maintains STUN servers (Session Traversal Utilities for NAT) across its infrastructure, identified by community packet analysis as hostnames like stun1.us.demonware.net. These handle NAT traversal for edge cases where direct connections between client and server require hole-punching through consumer routers.

Lobby Size History

The original Warzone launched with 150-player lobbies, with limited-time modes supporting up to 200. Warzone 2.0 on Al Mazrah controversially reduced this to 100. The count was later increased to 120, and as of recent updates, Raven Software has been intermittently testing 150-player lobbies again for standard Battle Royale playlists.

Network & Protocols

Warzone uses a server-authoritative networking model where the dedicated server runs the canonical game simulation. Clients send inputs and receive state updates. The IW engine uses a proprietary networking protocol over UDP for gameplay traffic.

The Tick Rate Problem

The most persistent technical criticism of Warzone is its server tick rate. Regular multiplayer runs at approximately 60 Hz. Warzone battle royale runs at approximately 20 Hz, sometimes dipping to lower values during high server load. Custom games and some large-format modes (like Ground War) have been measured as low as 12 Hz.

For context, Valorant runs at 128 Hz, Counter-Strike at 64 Hz, and Fortnite at 30 Hz. At 20 Hz, the server only processes game state updates 20 times per second. The time between updates (50ms) is long enough that players experience noticeable desync, particularly in close-quarters combat. Common symptoms include "super bullets" (multiple damage ticks arriving in a single frame because they were calculated across multiple server ticks but delivered together) and dying behind cover (the attacker's client saw you exposed on a stale server tick).

The tick rate is not static. Community analysis has shown that it fluctuates based on server load, with peak-hour tick rates dropping into the 12-16 Hz range on congested servers. The relationship between tick rate and perceived netcode quality is not linear, however. High tick rate with poor interpolation or prediction code can still feel bad, and low tick rate with good prediction can feel acceptable. The community consensus is that Warzone's prediction and interpolation code does not sufficiently compensate for the low base tick rate.

Why Not Higher Tick Rate?

Processing 150 players with full weapon ballistics, vehicle physics, and destructible environments at 60+ Hz is computationally expensive. Doubling the tick rate roughly doubles the server CPU cost per match, which across Warzone's global fleet of thousands of simultaneous matches translates to significant infrastructure spend. Activision has consistently chosen to keep BR tick rates lower than multiplayer, prioritizing match capacity over simulation fidelity. The community considers this a cost-saving measure rather than a technical necessity.

Latency Compensation

The IW engine implements standard latency compensation techniques: client-side prediction for movement, lag compensation (server rewinds to the shooter's perspective when validating hits), and interpolation between server state updates. The "Delta Ping" model attempts to place lobbies on the data center that minimizes the worst-case ping for all players in the lobby, rather than simply connecting everyone to their nearest server. This is a direct consequence of SBMM: tightening skill brackets sometimes requires pulling players from farther-away regions.

Data & Persistence

Demonware manages all persistent player data across the Call of Duty ecosystem. When a player unlocks a weapon, earns a camo, or levels up, Demonware's inventory services ensure that data persists across sessions and platforms. The unified Activision ID system links platform accounts (PSN, Xbox Live, Steam, Battle.net) to a single player profile, enabling cross-platform progression.

Cross-Title Integration

Warzone's data model is designed to persist across annual Call of Duty releases. When a new title launches (Modern Warfare III, Black Ops 6), Warzone integrates the new title's weapons, operators, and progression systems. This requires Demonware to maintain backward-compatible inventory schemas that can accommodate items from multiple game generations simultaneously. A weapon unlocked in Modern Warfare 2019 must still be accessible and functional in Warzone alongside weapons from Black Ops 6.

Killcam and Final Killcam

The killcam system records a server-side replay of the moments before a player's death. The server maintains a rolling buffer of recent game state, and when a kill occurs, the relevant window of state is packaged and sent to the victim's client for playback. Final killcams (the cinematic kill shown to all players at match end) are similarly generated from server-side state. This replay data is separate from the client-side theater/replay system and is lightweight enough to generate in real-time without impacting server tick performance.

Stats & Leaderboards

Demonware tracks per-match statistics (kills, damage, placement, cash earned, Gulag results) and aggregates them into career stats and leaderboards. Stats are scoped per-title and per-mode, with separate tracking for Warzone BR, Ranked Play, and each multiplayer mode. Leaderboard queries hit Demonware's backend directly rather than going through the game server.

Player Profiles & Gunsmith

Each player's loadout configurations (Gunsmith weapon builds, perk selections, equipment choices) are stored in Demonware's profile service. When a player purchases a loadout drop in-match, the Buy Station sends a request to the profile service to retrieve the player's saved loadout configuration. This is why there's a brief delay between purchasing a loadout drop and the crate actually appearing.

Scaling & Performance

Warzone's scaling model revolves around two challenges: supporting 150-player matches with acceptable netcode quality, and handling millions of daily active players across a global server fleet.

Match-Level Performance

A 150-player lobby represents substantially more server-side work than Fortnite's 100-player matches. Each additional player adds to the replication burden, hit registration calculations, and physics simulation workload. The IW engine handles this through aggressive level-of-detail networking: players far from each other receive reduced update rates, while close-proximity players get priority updates. This is similar in concept to Fortnite's Replication Graph but implemented within the IW engine's proprietary networking layer.

Map Streaming

Warzone's open-world maps are massive. Verdansk, Caldera, Al Mazrah, and Urzikstan are all significantly larger than standard 6v6 multiplayer maps. The IW engine streams map data on-demand rather than loading the entire map into memory at match start. An optional "on-demand texture streaming" feature downloads high-resolution textures from Activision's CDN during gameplay, trading internet bandwidth for reduced disk space and VRAM usage. This was particularly problematic during the Caldera launch, where players with slower connections experienced significant texture pop-in.

In 2024, Activision open-sourced the Caldera map dataset on GitHub for academic use, providing the geometry, collision volumes, and lighting data (though not textures) for one of the largest game environments ever shipped. The dataset demonstrates the complexity of Warzone's spatial data: millions of geometry elements with relationships between renderable objects, collision shapes, and lighting volumes.

SBMM Architecture

Skill-Based Matchmaking is handled by Demonware's Matchmaking+ service. The system evaluates players based on recent performance (community analysis suggests the last 5 matches weigh most heavily), with factors including K/D ratio, win rate, headshot percentage, and movement patterns. Several Activision patents describe matchmaking systems that go beyond simple skill ranking:

  • Patent US20160005270A1 describes a microtransaction engine that could match players with "marquee players" using premium items to influence purchase decisions
  • Patent US20190091581A1 describes matching based on player behavioral profiles, tracking how players respond to specific scenarios and matching similar playstyles
  • An internal "Deprioritize Skill Test" in 2024 loosened SBMM constraints, revealing that 90% of lower-skill players returned less frequently without protection, while high-skill players were more satisfied

Activision has stated these patents are R&D and not necessarily implemented, but the matchmaking system's behavior closely mirrors the patent descriptions. The SBMM system must balance skill brackets against latency requirements, and tightening skill constraints too aggressively forces the system to match players across wider geographic areas, increasing ping.

Technical Deep Dives

The Gulag

The Gulag is a respawn mechanic unique to Warzone. When eliminated for the first time in a match, players are sent to a separate arena for a 1v1 duel. The winner respawns back into the battle royale; the loser's only hope is a teammate buying them back at a Buy Station.

Server-side, the Gulag runs as a sub-instance within the same match server process. It is not a separate server. The main BR simulation continues while Gulag fights happen in parallel. The Gulag arena exists in the same world space but is isolated so players cannot interact with the main game. When a player dies and enters the Gulag queue, the server tracks their position in the queue and spawns them into the arena when their turn arrives. Spectating other Gulag fights happens through the same server replication channel, with the server sending Gulag arena state to queued players.

In Warzone 2.0, the Gulag was reworked into a 2v2 format with random pairing, loot spawns on the ground instead of fixed loadouts, and an AI "Jailer" enemy that both teams could cooperate to defeat for a mutual escape. This significantly increased the Gulag's server-side complexity because the system now had to manage four players, AI pathfinding, loot spawns, and a cooperation detection system within the sub-instance.

Buy Station Economy

Warzone's in-match economy is entirely server-authoritative. Cash is tracked per-player on the server. When a player interacts with a Buy Station, the client sends a purchase request and the server validates the player's cash balance before deducting the cost and spawning the purchased item. Loadout drops require a server-side call to retrieve the player's saved loadout configuration from Demonware's profile service.

Buy Station pricing has been tuned repeatedly across seasons. Self-revive kits, UAVs, loadout drops, and teammate redeployment costs all shift to change the strategic meta. The server enforces stock limits (some items are limited to 2 purchases per station) and applies perk-based discounts (Hardline provides 25% off). The economy creates a cash flow loop: players earn cash through looting, eliminations, and completing Contracts (bounties, recon missions, supply runs), then spend it on combat advantages and teammate revivals.

RICOCHET Anti-Cheat In Depth

RICOCHET is a five-layer anti-cheat system that Activision developed internally starting in 2021:

Layer 1: TPM 2.0 and Secure Boot. The newest layer, added with Black Ops 7 and retroactively applied to Warzone. When the game launches, the system verifies that the PC's Trusted Platform Module reports a clean boot chain. Platform Configuration Registers (PCRs) record the system's startup state, and any modification to the boot process (like disabling Secure Boot to load unsigned drivers) flags the system as untrusted. The TPM also provides a hardware-bound unique identifier that cannot be spoofed, enabling hardware bans that survive OS reinstalls.

Layer 2: Kernel Driver. The RICOCHET kernel driver operates at Ring 0 (the same privilege level as graphics card and OS drivers). It monitors all processes attempting to interact with the Call of Duty game process, detecting unauthorized memory reads/writes, injected DLLs, and hooking attempts. Unlike Valorant's Vanguard, the RICOCHET driver only activates when the game launches and shuts down when the game closes. It is installed as part of the game and removed when the game is uninstalled.

Layer 3: Server-Side Machine Learning. Behavioral analysis models run on the server side, evaluating gameplay telemetry for anomalies: statistically impossible aim accuracy, inhuman reaction times, movement patterns consistent with speed hacks, and damage output that exceeds weapon capabilities. These models flag players for review and feed into the mitigation system.

Layer 4: In-Game Mitigations. Rather than immediately banning detected cheaters, RICOCHET deploys active countermeasures to collect more data while neutralizing the cheater's impact. Damage Shield makes legitimate players immune to cheaters' bullets. Cloaking makes legitimate players invisible to the cheater. Disarm removes the cheater's weapons. Hallucinations spawns fake decoy players that only the cheater can see. These mitigations keep the cheater in-game for extended data collection while protecting other players.

Layer 5: Account Enforcement. Ban waves are issued daily and in large batches. Phone verification links accounts to phone numbers (raising the cost of creating throwaway accounts). Shadow lobbies route flagged players into separate matchmaking pools with higher latency and longer queue times. Third-party hardware detection identifies devices like Cronus adapters. Since Black Ops 6 launched, RICOCHET has banned over 228,000 accounts.

Audio Occlusion

Warzone's audio system uses a hybrid client-server model. The IW engine performs audio occlusion calculations on the client side using raycasts against the game's geometry to determine which surfaces block or attenuate sound. Footstep audio, gunfire, and vehicle sounds are spatially positioned based on the client's local representation of other players' positions, which means audio accuracy is directly tied to the server's update rate. At 20 Hz, there is up to 50ms of positional staleness in audio cues, which experienced players can notice when trying to pinpoint enemy positions through walls.

Architecture Evolution

2003-18 IW Engine lineage begins. id Tech 3 forks into the first Call of Duty engine. Through 15+ iterations across yearly titles, the engine evolves from a corridor shooter framework into an open-world capable rendering and networking platform. Demonware (acquired 2007) becomes the backbone for all online features.
2019 Modern Warfare (2019) ships on IW 8.0. Major engine rewrite with photogrammetry, a new animation system, and 64-player Ground War mode. The engine's ability to handle larger player counts lays the groundwork for Warzone. Battle(non)sense's netcode analysis reveals variable tick rates across modes.
2020 Warzone launches (March 10). 150 players on Verdansk. 15 million players in the first 4 days. 50 million within the first month. The Gulag 1v1 respawn mechanic differentiates it from other BRs. SBMM controversy begins immediately. Tick rate measured at 20-24 Hz for BR.
2021 RICOCHET Anti-Cheat announced (October). Kernel-level driver deploys first in Asia-Pacific, then globally. Cheating had become an existential problem for Warzone, with content creators and competitive players abandoning the game. Verdansk is replaced by Caldera (Pacific map).
2022 Warzone 2.0 launches on Al Mazrah. New map, 2v2 Gulag, DMZ extraction mode. Player count controversially reduced to 100. RICOCHET adds Damage Shield, Cloaking, and Disarm mitigations. The in-game mitigation philosophy keeps cheaters playing for data collection rather than immediately banning them.
2023 MW3 integration and Urzikstan map. Lobby count returns to 120. Activision publishes SBMM research data and conducts the "Deprioritize Skill Test." The Gulag returns to 1v1 with three map variants and dynamic events. Buy Station economy rebalanced toward original Warzone pricing.
2024 Verdansk returns, Caldera dataset open-sourced. 150-player lobbies tested. RICOCHET adds Hallucinations mitigation and third-party hardware detection (Cronus). TPM 2.0 and Secure Boot requirements added for Black Ops 7/Warzone. Demonware issues 228,000+ bans post-BO6 launch.
2025 RICOCHET adds TPM 2.0 hardware identity verification. Platform Configuration Registers and hardware-bound bans make it significantly harder for repeat offenders to create new accounts. Demonware infrastructure improvements address rubber-banding and packet loss in Ranked Play.
